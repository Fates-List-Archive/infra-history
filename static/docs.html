Webhook Documentation

There are many types of webhooks in Fates List, however you MUST select Vote Webhook for votes unless you are using Fates Client or another library
Vote Webhook Your webhook will be sent in this format: id: String -> User ID votes: Integer -> Votes The webhook will have a Authorization header with your API Token. The Test API Token (so you can try out the webhook) is eb83da1b-1bbf-44e6-a938-5545bcc2934f. This only applies to webhooks and not for other non webhook related things

WebSocket Documentation

Fates List offers a WebSocket API at /api/ws for getting real time stats on your bot. This is the only way to get view events (and potentially other events)

Upon attempting to connect to /api/ws, you will get a JSONs with the following basic syntax:


  

    Key
    Description
    Type
  

  

    payload
    This is a payload which is essentially what is being sent to you. This can be a IDENTITY, STATUS, KILL_CONN or EVENTS but this may change in the future
    String
  

  

    type
    This is the payload type which is how you are supposed to respond or extra information about the payload such as NO_AUTH or API_TOKEN.
    String
  




{"payload": "IDENTITY", "type": "API_TOKEN"}
